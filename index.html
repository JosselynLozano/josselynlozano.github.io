Comandos Git

Git clone

Este comando sirve para "clonar" o "descargar" el codigo fuente desde un repositorio.

Se usa de la siguiente manera:

git clone URL del repositorio que se desea clonar

Git branch

Se utliza para crear una rama. La ramas son importantes para que varios desarrolladores puedan trabajar simultaneamente en el mismo proyecto.

Se usa de la siguiente manera:

git branch nombre de la rama

Git branch --list

Se utliza para listar una rama.

Git branch -d

Se utliza para borrar una rama.

Git checkout

Principalmente se ocupa para cambiarte de una rama a otra.

Se usa de la siguiente manera:

git checkout nombre de la rama

Git status

Este comando nos da toda la información necesaria sobre la rama actual.

Se usa de la siguiente manera:

git status

Git add

Cuando se crea, se modifica o se elimina un archivo, estos cambios suceden en local hasta que se usa este comando para incluir los cambios del archico en el siguiente commit.
Se usa de la siguiente manera:

git add Nombre del archivo

Si se quiere agregar todo de una vez:

git add -A

Git commit

Este comando establece un punto de control en el proceso de desarrollo al cual se puede volver más tarde si es necesario.

Se usa de la siguiente manera:

git commit mensaje de confirmación

cat

Se utiliza para crear un archivo y agregarle texto.

Se usa de la siguiente manera:

cat filename.txt

nano

Se utiliza para abrir y editar un archivo de texto.

Se usa de la siguiente manera:

nano filename.txt

Git push

Después de haber hecho todos los cambios, se debe enviar los cambios al servidor remoto. Git push envía los commits al repositorio remoto.

Se usa de la siguiente manera:

git push

Git pull

Se utiliza para recibir actualizaciones del repositorio remoto.

Se usa de la siguiente manera:

git pull

Git merge

Este comando se utiliza para fucionar la rama con la rama master.

Se usa de la siguiente manera:

Primero, se debe cambiar a la rama master:

git checkout main

Y ahora si, se puede fucionar la rama:

git merge nombre de la rama
